# This is an example Devbook App

When adding a new field to a model with existing data, you can either:

- Use the `?` operator to mark the field as nullable
- Provide a default value using the `@default()` attribute function

---

On the left is a Prisma schema (`prisma/schema.prisma`) file.

<Check checkID="Add likes column">

Update the `Post` model by adding a `likes` field with a default value of 0.

<CodeBlock
  title="schema.prisma"
  lang="prisma"
>
{`likes     Int      @default(0)`}
</CodeBlock>

> **HINT**: You can hit <kbd>CMD</kbd>/<kbd>CTRL</kbd> + <kbd>S</kbd> to invoke the auto formatting in the  Prisma schema.

Your `Post` model should resemble the following after adding the `likes` field:

<CodeBlock
  title="schema.prisma"
  lang="prisma"
>
{`model Post {
  id        Int      @id @default(autoincrement())
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  title     String   @db.VarChar(255)
  content   String?
  published Boolean  @default(false)
  likes     Int      @default(0)

  authorId  Int?
  author    User?     @relation(fields: [authorId], references: [id])
}`}
</CodeBlock>

</Check>
