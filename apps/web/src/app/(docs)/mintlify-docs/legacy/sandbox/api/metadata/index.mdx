title: Sandbox metadata

<Callout icon="circle-exclamation" color="#ff8800" iconType="regular">You are reading a legacy (pre v1.0) document.</Callout>


You can assign metadata to your sandbox. This metadata will be available if you call `Sandbox.list()`. This can be useful in a situation where each of your users have dedicated sandbox and the time between the steps of the integration is long. You can use the metadata to store the user id and reconnect to it later.

<CodeGroup>
```js JavaScript & TypeScript highlight={5}
import { Sandbox } from 'e2b'

const sandbox = await Sandbox.create({
  template: 'base',
  metadata: { userID: 'uniqueID' }
})
// Keep the sandbox alive for 60 seconds
await sandbox.keepAlive(60_000)
// You can even close the script

// Later, can be even from another process
// List all running sandboxes
const paginator = Sandbox.list({ query: {state: ['running']}})
const runningSandboxes = await paginator.nextItems()
// Find the sandbox by metadata
const found = runningSandboxes.find(s => s.metadata?.userID === 'uniqueID')
if (found) {
  // Sandbox found, we can reconnect to it
  const sandbox = await Sandbox.reconnect(found.sandboxID)
} else {
  // Sandbox not found
}

await sandbox.close()
```
```python Python highlight={6}
from e2b import Sandbox, SandboxState
from e2b.sandbox.sandbox_api import SandboxQuery

sandbox = Sandbox(
    template="base",
    metadata={"user_id": "uniqueID"},
)

# Keep the sandbox alive for 60 seconds
sandbox.set_timeout(60)
# You can even close the script

# Later, can be even from another process
# List all running sandboxes
paginator = Sandbox.list(query=SandboxQuery(state=[SandboxState.RUNNING]))
running_sandboxes = paginator.next_items()

# Find the sandbox by metadata
for running_sandbox in running_sandboxes:
    if running_sandbox.metadata.get("user_id", "") == "uniqueID":
        sandbox = Sandbox.connect(running_sandbox.sandbox_id)
        break
else:
    # Sandbox not found
    pass

sandbox.close()
```
