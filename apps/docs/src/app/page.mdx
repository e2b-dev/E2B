# Sandbox for AI Apps & Agents

The E2B Sandbox is a secure way to run your AI app. It is a long-running cloud environment where you can let any LLM (GPTs, Claude, local LLMs, etc) use tools exactly like you would do locally. {{ className: 'lead' }}


## How Sandboxes Work
Under the hood, the sandboxes are general purpose machines powered by Ubuntu. Learn more about all features and comparison to other services [here](/sandbox/overview).


## What Can You Build with Sandboxes
The sandboxes can do majority of the things your AI app or agent can do locally. A few popular use cases are:

- **AI data analysis**
- **Code interpreter**
- **Run LLM-generated code**
- **Coding copilots**
- **AI-powered browsers in the cloud**

## Quick start

### 1. Create an account
<Button variant="textLink">Sign in</Button> with your GitHub acccount.

### 2. Get your API key
After sign in, get [your API key](/getting-started/api-key) and set it in the `E2B_API_KEY` environment variable.

### 3. Install the E2B SDK
We offer both JS and Python SDK.

<CodeGroup isRunnable={false}>
```bash {{ language: 'js' }}
npm install e2b
```

```bash {{ language: 'python' }}
pip install e2b
```
</CodeGroup>

### 4. Start up the sandbox
Sandbox is an isolated cloud environment for your AI app, agent, assistant, etc.

<CodeGroup isRunnable={false}>
```js {{ language: 'js' }}
// 1. Import E2B
import { Sandbox } from 'e2b'

// 2. Get your Sandbox session
const sandbox = await Sandbox.create()

// 3. Close the sandbox once done
await sandbox.close()
```

```python {{ language: 'python' }}
# 1. Import E2B
from e2b import Sandbox

# 2. Get your Sandbox session
sandbox = Sandbox()

# 3. Close the sandbox once done
sandbox.close()
```
</CodeGroup>
